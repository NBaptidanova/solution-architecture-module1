# Архитектурное видение проекта Diamond Universe


## Задача:
Разработать систему для продажи алмазов с помощью онлайн аукциона для компании «Diamond Universe». Решение сделает покупку алмазов доступной без выезда эксперта покупателя, что должно увеличить объем продаж и повысить выручку компании.  

## Контекст:
Во время пандемии 2020г продажи алмазов снизились почти на 50%. Ключевыми факторами послужили закрытие ритейла, остановка добычи и логистики, массовые отмены мероприятий. Рынок начал восстанавливаться благодаря переходу его участников в онлайн. Компания Diamond Universe также приняла решение проводить онлайн аукционы. 

*Шаги нового бизнес процесса, который нужно автоматизировать:*
  
1. Отбор алмазов для продажи онлайн в центральном офисе.
2. Перевозка алмазов в лабораторию, их оцифровка, и возврат обратно в офис. 
3. Публикация данных для покупателей, на основе которых будут приниматься решения. 
4. Проведение онлайн-аукциона. 
5. Отправка камней победителям торгов из офиса.  

**Текущее состояние:** В компании есть системы, которые частично автоматизируют процесс проведения оффлайн аукционов: 
- Основной бизнес: информационные системы (ИС) «Алмазы» и «Аукцион» 
- Внутренние обеспечивающие системы: ИС «Бухгалтерия» и ИС «Логистика».
- Внешние обеспечивающие системы: ИС «Лаборатория» (новая) и облачные сервисы

**Целевое состояние:** В комании есть система полностью автоматизирующая бизнес-процесс подготовки и проведения онлайн-аукциона: 
- Все шаги процесса автоматизированы с доработкой имеющихся систем
- Есть возможность отслеживать план и статус подготовки и проведения аукциона
- Все интеграции автоматизированы
- Для передачи конфиденциальной информации используются выделенные каналы
- Используемые облачные сервисы находятся на серверах компании
  
**Ограничения:**
Срок проведения первого онлайн аукциона - через 2 месяца.  


## Описание оптимального решения
 Чтобы провести аукцион в срок через 2 месяца требуется разделить проект на промежуточное и целевое решение.
 
 **Промежуточное решение**
 
 Задачи:
 - частичная автоматизация подготовки данных аукциона (разработать шаблоны и подготовить данные онлайн аукциона (Excel))
 - использовать существующие ИС для планирования, логистики и проведения аукциона (с ручной обработкой/передачей данных)
 - использовать внешний облачный сервис для публикации данных для клиентов аукциона (заключить контракт с провайдером частного облака)

 Требуемые ресурсы:
 - 2 разработчика, бизнес-эксперт для консультаций, юрист для заключения контракта 

Сроки реализации: 2 месяца

**Целевое решение**
 
 Задачи:
 - полная автоматизация подготовки данных аукциона (данные между системами передаются по интеграции)
 - существующие ИС доработаны и обеспечивают автоматическое планирование, логистику и проведение аукциона (доработан функционал систем, данные между системами передаются по интеграции)
 - облачный сервис для публикации данных для клиентов размещен на сервере компании (закупка оборудования, развертывание сервиса, миграция данных)
 - для передачи конфиденциальной информации используются защищенные каналы (авторизация, аутентификация, шифрование данных)

 Требуемые ресурсы:
 - Команда разработки, инженер DevOps, специалист поддержки, бизнес-эксперты для консультаций, юрист для проведения закупки оборудования 

Сроки реализации: 12 месяцев
